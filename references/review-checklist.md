# 评估清单

## 去重

- 统一成 `owner/repo`。
- 去掉 `.git`。
- 去掉 `/tree/...`、`/blob/...` 和 README 子路径。
- 身份识别时忽略查询参数。

## 分级

当仓库能改善用户自己的工作流时，用 `use`。

当仓库有清楚的故事、演示或受众痛点时，用 `publish`。

当仓库有潜力，但还需要安装或运行验证时，用 `test`。

当仓库重复、过时、过窄或不相关时，用 `archive`。

## 安装验证门槛

一次只测一个仓库。

测试前先记录：

- 为什么需要测
- 预期能看到什么结果
- 安装命令来自哪里
- 回滚或清理说明

同一种错误连续失败两次后就停下来诊断。
